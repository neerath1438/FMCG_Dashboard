# FMCG டேஷ்போர்டு: பேக்-எண்ட் தொழில்நுட்ப விவரக்குறிப்பு (Technical Specification)

இந்த ஆவணம் FMCG டேஷ்போர்டின் பேக்-எண்ட் செயல்பாடுகளை முழுமையாக விளக்குகிறது. இது ஒரு டெவலப்பர் அல்லது AI மாடல் இந்த சிஸ்டத்தை எளிதில் புரிந்துகொள்ள உதவும் வகையில் உருவாக்கப்பட்டுள்ளது.

---

## 1. சிஸ்டம் கட்டமைப்பு (System Architecture)
இந்த சிஸ்டம் **FastAPI** மற்றும் **MongoDB** கொண்டு உருவாக்கப்பட்டுள்ளது.

*   **மொழி**: Python 3.10+
*   **தகவல் தளம்**: MongoDB
*   **AI என்ஜின்**: Azure Claude (முதன்மையானது) மற்றும் Azure OpenAI (மற்றுமொரு ஆதரவு).
*   **தரவு கையாளுதல்**: Pandas (Batch Processing)

---

## 2. தகவல் தள அமைப்பு (Database Schema)
`fmcg_mastering` என்ற டேட்டாபேஸில் கீழ்க்கண்ட முக்கியமான கலெக்ஷன்கள் (Collections) உள்ளன:

| கலெக்ஷன் பெயர் | நோக்கம் | முக்கிய புலங்கள் (Fields) |
| :--- | :--- | :--- |
| `raw_data` | பதிவேற்றப்பட்ட அசல் கோப்புகளின் சேமிப்பு. | அசல் காலம்கள் + `sheet_name`. |
| `single_stock_data` | **Flow 1**-ன் முடிவு. UPC அடிப்படையில் சுருக்கப்பட்ட தரவு. | `UPC`, `ITEM`, மொத்த விற்பனை மதிப்புகள். |
| `master_stock_data` | **Flow 2**-ன் முடிவு. AI மூலம் சீரமைக்கப்பட்ட தயாரிப்புகள். | `brand`, `flavour`, `size`, `merge_rule`. |
| `7-eleven_data` | 7-Eleven-ன் செறிவூட்டப்பட்ட தரவுகள். | `ArticleCode`, `GTIN`, `7E_Variant`, `7E_MPack`. |
| `LLM_CACHE_STORAGE` | AI முடிவுகளைச் சேமிக்கும் தற்காலிக நினைவகம் (Cache). | `item` (key), `result` (JSON). |
| `7eleven_extra_items` | இடைவெளி பகுப்பாய்வு (Gap Analysis) முடிவுகள். | `UPC`, `Match_Level`, `Main_UPC`. |

## 3. தரவு செயலாக்க நிலைகள் (Data Processing Flows)

### நிலை 1: Flow 1 (தரவு உள்ளீடு மற்றும் தொடக்க நிலை சீரமைப்பு)
1.  **காலம் கண்டறிதல்**: `UPC`, `MARKETS`, `MPACK` போன்ற முக்கிய காலம்களை தானாகவே அடையாளம் காணும்.
2.  **UPC அடிப்படையிலான குழுவாக்கம்**: `UPC + Market + MPack + Facts` அடிப்படையில் தரவு பிரிக்கப்படும்.
    *   **விதி**: ஒரே பார்கோடு ஆனால் வெவ்வேறு பேக்கிங் (`X1` vs `X12`) இருந்தால் அவை தனித்தனியாகவே வைக்கப்படும்.
3.  **மதிப்பு கூட்டல்**: ஒரு குழுவில் உள்ள அனைத்து வரிகளின் விற்பனை மதிப்புகளும் (Sales Values) ஒன்றாகக் கூட்டப்படும்.

### நிலை 2: Flow 2 (AI-மூலம் தயாரிப்புகளைச் சீரமைத்தல்)
இது AI-ஐப் பயன்படுத்தி தயாரிப்புகளை ஒழுங்குபடுத்தும் நிலை.

1.  **AI பண்பு பிரிப்பு**: தயாரிப்பு பெயரிலிருந்து `Brand`, `Flavour`, `Size`, `Variant` ஆகியவற்றை AI பிரித்தெடுக்கும்.
    *   **முக்கிய விதி**: "15g X 12" போன்ற அளவுகளை AI பெருக்கவோ மாற்றவோ கூடாது (Strict Size Extraction).
2.  **அறிவுசார் குழுவாக்கம் ( मास्टरिंग )**:
    *   AI-ன் நம்பிக்கை அதிகமாக இருந்தால் (Confidence >= 0.92), AI கொடுத்த பண்புகள் அடிப்படையில் தயாரிப்புகள் இணைக்கப்படும்.
    *   சந்தேகமாக இருந்தால், தவறான இணைப்பைத் தவிர்க்க "Simple Clean" முறை பயன்படுத்தப்படும்.
3.  **பாதுகாப்பு விதிகள் (Hard Guards)**:
    *   வெவ்வேறு சுவைகளை (உதாரணமாக: Vanilla vs Chocolate) தவறுதலாகக் கூட இணைக்காதவாறு பாதுகாப்பு விதிகள் உள்ளன.
4.  **தலைமைத் தயாரிப்பு தேர்வு**: இணைக்கப்பட்ட பொருட்களில், எது அதிக விற்பனையாகிறதோ அந்தப் பெயரே "மாஸ்டர் பெயராக" (Master Name) வைக்கப்படும்.

## 4. இடைவெளி பகுப்பாய்வு (Gap Analysis Logic)
Nielsen சந்தை தரவுகளை 7-Eleven தரவுகளுடன் ஒப்பிடும் முறை:

*   **நிலை 1**: பார்கோடு (UPC) சரியாக இருந்தால் (`Nielsen.UPC == 7Eleven.GTIN`) அவை மேட்ச் செய்யப்படும்.
*   **நிலை 2**: பார்கோடு இல்லாவிட்டாலும், பிராண்ட், சுவை மற்றும் அளவு (±5g வித்தியாசம்) சரியாக இருந்தால் அவை மேட்ச் செய்யப்படும்.

---

## 5. இறுதி இலக்கு மற்றும் மூலோபாயம் (Final Target Strategy) 🎯

நமது சிஸ்டத்தின் இறுதி இலக்கு வெறும் தரவுகளைச் சீரமைப்பது மட்டுமல்ல, விற்பனையை அதிகரிப்பதற்கான வழிகளைக் காட்டுவதுதான்.

### நம்முடைய இலக்கு (Target Goals):
1.  **MPack வாய்ப்புகள்**: சந்தையில் `X12` அல்லது `X24` போன்ற பெரிய பேக்குகள் வெற்றிகரமாக ஓடிக்கொண்டிருக்கும் ஆனால் 7-Eleven-ல் `X1` மட்டும் இருந்தால், அந்தப் பெரிய பேக்குகளை உடனே விற்பனைக்குக் கொண்டுவருவது.
2.  **விடுபட்ட தயாரிப்புகள் (Assortment Gaps)**: மற்ற இடங்களில் மிக அதிக விற்பனையாகும், ஆனால் இன்னும் 7-Eleven அலமாரிகளில் இடம் பெறாத தயாரிப்புகளை (SKUs) உடனடியாகப் பட்டியலிடுவது.
3.  **விலை உத்தி**: மொத்தமாக விற்பனை செய்யும்போது கிடைக்கும் லாபத்தையும் (Bulk Price Advantage), சந்தை விலையுடன் ஒப்பிட்டுப் சிறந்த விலையை நிர்ணயம் செய்வது.
4.  **AI துல்லியம்**: மனிதத் தலையீடு இல்லாமலேயே 99% துல்லியத்துடன் தரவுகளை மேலாண்மை செய்வது.

---
