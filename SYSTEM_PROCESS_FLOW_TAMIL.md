# FMCG Dashboard: முழுமையான கணினி செயல்முறை விளக்கம் (End-to-End System Process Flow)

இந்த ஆவணம், FMCG டேஷ்போர்டு எவ்வாறு தரவுகளை (Data) கையாள்கிறது என்பதைப் படிப் படியாக விளக்குகிறது. எக்செல் பதிவேற்றம் முதல் AI பயன்பாடு மற்றும் விற்பனை வாய்ப்புகளைக் கண்டறிவது வரை அனைத்தும் இதில் அடங்கும்.

---

## 1. நிலை 1: தரவுப் பதிவேற்றம் மற்றும் ஆரம்பகட்ட சுத்திகரிப்பு (Flow 1)
**நோக்கம்**: பல்வேறு வடிவங்களில் உள்ள எக்செல் (Excel) கோப்புகளைச் சீரமைத்து, ஒரு நிலையான வடிவத்திற்கு மாற்றுவது.

*   **படி 1.1: கோப்பு பதிவேற்றம்**: சிஸ்டம் `.xlsx`, `.xls` அல்லது `.csv` கோப்புகளை ஏற்றுக்கொள்கிறது. இது தானாகவே அந்த கோப்பு நீல்சன் (Nielsen) அறிக்கையா அல்லது 7-Eleven தரவா என்பதைக் கண்டறியும்.
*   **படி 1.2: நெடுவரிசை பொருத்தம் (Column Mapping)**: வெவ்வேறு பெயர்களில் இருக்கும் நெடுவரிசைகளை (உதாரணமாக "UPC", "EAN", அல்லது "GTIN") சிஸ்டம் தானாகவே அடையாளம் கண்டு ஒருங்கிணைக்கிறது.
*   **படி 1.3: அசல் தரவு சேமிப்பு**: பதிவேற்றப்பட்ட அசல் கோப்பில் உள்ள அனைத்து வரிசைகளும் எந்த மாற்றமும் இன்றி `raw_data` என்ற பகுதியில் தணிக்கைக்காகச் சேமிக்கப்படுகிறது.
*   **படி 1.4: தரவு ஒருங்கிணைப்பு (Grouping & Aggregation)**:
    *   ஒரே மாதிரியான UPC, மார்க்கெட் (Market), மற்றும் பேக் வகை (Pack) கொண்ட தரவுகள் ஒன்றாகச் சேர்க்கப்படுகின்றன.
    *   மாதாந்திர விற்பனை மதிப்புகள் (Sales Value) மற்றும் எண்ணிக்கைகள் (Units) கூட்டப்படுகின்றன.
*   **படி 1.5: இறுதிச் சேமிப்பு**: சுத்திகரிக்கப்பட்ட இந்தத் தரவுகள் `single_stock_data` என்ற பகுதியில் சேமிக்கப்படுகின்றன.

## 2. நிலை 2: AI மூலம் தயாரிப்புகளைச் சீரமைத்தல் (Flow 2)
**நோக்கம்**: பெயர்களில் உள்ள பிழைகள் மற்றும் மாறுபாடுகளைச் செயற்கை நுண்ணறிவு (AI) மூலம் சரிசெய்து, ஒரே மாதிரியான தயாரிப்புகளை ஒரு குழுவாக இணைப்பது.

*   **படி 2.1: பண்புகளைப் பிரித்தல் (AI Extraction)**:
    *   தயாரிப்பின் பெயர்களை (உதாரணமாக: `OAT KRUNCH CHOCO 156G X 12`) AI ஆராய்கிறது.
    *   அதிலிருந்து பிராண்ட் (Brand), தயாரிப்பு வகை (Product Line), சுவை (Flavour), எடை (Size), மற்றும் பேக் வகை (Pack Type) ஆகியவற்றைத் துல்லியமாகப் பிரிக்கிறது.
*   **படி 2.2: புத்திசாலித்தனமான குழுவாக்கம் (Clustering)**:
    *   தயாரிப்புப் பெயர்களை நேரடியாக ஒப்பிடாமல், AI பிரித்தெடுத்த பண்புகளை வைத்து (Brand + Flavour + Size) ஒரே மாதிரியான பொருட்கள் ஒன்றாக இணைக்கப்படுகின்றன.
*   **படி 2.3: பிழை திருத்தம் (Fuzzy Matching)**: எழுத்துப் பிழைகள் இருந்தாலும் (உதாரணமாக: `CADBURY` மற்றும் `CDBRY`), சிஸ்டம் அவற்றை அடையாளம் கண்டு ஒன்றாக இணைக்கும்.
*   **படி 2.4: பாதுகாப்புத் தணிக்கை (Hard Guards)**:
    *   தவறான இணைப்புகளைத் தவிர்க்க ஒரு தானியங்கி பாதுகாப்பு முறை உள்ளது. இது வெவ்வேறு சுவைகள் (உதாரணமாக: Strawberry மற்றும் Chocolate) தவறுதலாகக் கூட ஒன்றாக இணையாமல் பார்த்துக் கொள்கிறது.
*   **படி 2.5: முதன்மைப் பெயர் தேர்வு (MAT Leader)**: ஒரு குழுவில் அதிக விற்பனையாகும் தயாரிப்பின் பெயரே அந்த முழு குழுவிற்கும் "முதன்மைப் பெயராக" (Master Name) வைக்கப்படுகிறது.

## 3. நிலை 3: விற்பனை இடைவெளி ஆய்வு (Mapping Analysis)
**நோக்கம்**: பொதுச் சந்தையில் (Nielsen) விற்கும் பொருட்களையும், 7-Eleven கடைகளில் விற்கும் பொருட்களையும் ஒப்பிட்டுப் பார்ப்பது.

*   **படி 3.1: தரவுச் சீரமைப்பு**: இரண்டு பக்கமும் உள்ள பெயர்கள் மற்றும் எடைகள் ஒரு நிலையான வடிவத்திற்கு (உதாரணமாக: `300gm` என்பது `300.0` என்று) மாற்றப்படுகின்றன.
*   **படி 3.2: நிலை 1 - நேரடிப் பொருத்தம் (UPC Match)**: முதற்கட்டமாக, UPC குறியீடு சரியாகப் பொருந்தும் பொருட்கள் இணைக்கப்படுகின்றன.
*   **படி 3.3: நிலை 2 - பண்புகள் பொருத்தம் (Attribute Match)**:
    *   UPC குறியீடு இல்லையென்றால், பிராண்ட், சுவை, எடை ஆகியவற்றைக் கொண்டு சிஸ்டம் தேடுகிறது.
    *   எடையில் 5 கிராம் (±5g) வரை வித்தியாசம் இருந்தாலும் இது சரியாக அடையாளம் காணும்.
*   **படி 3.4: மாற்றுப் பெயர்கள் (Synonyms)**: `ORIGINAL`, `PLAIN`, `REGULAR` போன்ற ஒரே பொருள் தரும் வெவ்வேறு சொற்களைச் சிஸ்டம் புரிந்து கொண்டு இணைக்கிறது.
*   **படி 3.5: இருவழி ஆய்வு (Bi-Directional Gaps)**:
    *   **7-Eleven Gaps**: 7-Eleven-ல் இருந்து மார்க்கெட்டில் இல்லாத பொருட்கள்.
    *   **Market Heroes (Master Gaps)**: மார்க்கெட்டில் (Master Stock) மிக அதிகமாக விற்று, ஆனால் 7-Eleven-ல் இல்லாத பொருட்கள். 7-Eleven தனது விற்பனையை அதிகரிக்க இவை தான் மிக முக்கியமான இலக்குகள்.

## 4. நிலை 4: வணிக நுண்ணறிவு மற்றும் கவரேஜ் (Reporting & Coverage)
**நோக்கம்**: தரவுகளைக் கொண்டு விற்பனையை அதிகரிப்பதற்கான வழிகளைக் காட்டுவது மற்றும் சந்தைப் பங்கை அளவிடுவது.

*   **படி 4.1: எண்ணிக்கை கவரேஜ் (Item Coverage)**: 7-Eleven பொருட்களில் எத்தனை சதவீதம் மார்க்கெட் தரவுகளுடன் இணைந்துள்ளது என்பதைக் கணக்கிடுகிறது.
*   **படி 4.2: சந்தை மதிப்பு கவரேஜ் (Value Coverage)**: இது மிக முக்கியமான அளவீடு. ஒட்டுமொத்த மார்க்கெட் விற்பனை மதிப்பில் (Value) 7-Eleven தற்போது எத்தனை சதவீதத்தை வைத்துள்ளது என்பதைக் காட்டுகிறது.
*   **படி 4.3: முன்னுரிமை அளித்தல்**: "Market Heroes" பொருட்கள் அவற்றின் விற்பனை மதிப்பின் அடிப்படையில் வரிசைப்படுத்தப்படுகின்றன. எந்தப் பொருட்களை அறிமுகப்படுத்தினால் அதிக லாபம் கிடைக்கும் என்று இது 7-Eleven-க்குத் துல்லியமாகக் காட்டும்.
*   **படி 4.4: ஒருங்கிணைந்த அறிக்கை (Unified Export)**: 7-Eleven தரவுகள் மற்றும் மார்க்கெட் இடைவெளிகள் (Market Heroes) ஆகிய இரண்டையும் ஒரே எக்செல் கோப்பில் வழங்குகிறது.

## 5. பின்னிணைப்பு: சரிபார்ப்பு குவெரிகள் (QA Verification Queries)
கிளையண்ட் மீட்டிங்கின் போது இந்த குவெரிகளை **NoSQLBooster**-ல் பயன்படுத்தி நமது முன்னேற்றத்தை நிரூபிக்கலாம்.

### Q1: ஒட்டுமொத்த மேப்பிங் நிலை (Overall Summary)
மேட்ச் ஆனவை மற்றும் விடுபட்டவை (Gaps) பற்றிய சுருக்கம்.
```javascript
db.mapping_results.aggregate([
  { $group: { _id: "$Match_Level", count: { $sum: 1 } } },
  { $project: { "Status": "$_id", "Count": "$count", "Percentage": { $round: [{ $multiply: [{ $divide: ["$count", 3220] }, 100] }, 2] } } }
])
```

### Q2: யூனிக் ஆர்டிகல் கோடு கவரேஜ் (Unique ArticleCode Coverage)
மிஸ்ஸிங் டேட்டா சரிசெய்யப்பட்டதை உறுதி செய்ய.
```javascript
db.mapping_results.distinct("ArticleCode", { "Match_Level": { $in: ["LEVEL_1", "LEVEL_2"] } }).length
```

### Q3: பார்-கோடு மேட்ச் நிரூபணம் (Suffix Matching Proof)
8-இலக்க மற்றும் 13-இலக்க பார்-கோடுகள் வெற்றிகரமாக இணைக்கப்பட்டதைக் காட்ட (8% Gap Fix).
```javascript
db.mapping_results.find({ 
  "Match_Type": "Exact/Flexible UPC Match", 
  $expr: { $ne: [ { $strLenCP: "$GTIN" }, { $strLenCP: { $ifNull: ["$Main_UPC", ""] } } ] } 
}).limit(10)
```

### Q4: டாப் மார்க்கெட் ஹீரோக்கள் (Top Market Heroes)
7-Eleven அறிமுகப்படுத்த வேண்டிய மிக முக்கியமான பொருட்கள்.
```javascript
db.mapping_results.find({ "Match_Level": "MARKET_HERO" })
  .sort({ "Matched_MAT": -1 })
  .limit(20)
```
